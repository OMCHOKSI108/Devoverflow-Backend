# INSTALL THESE PACKAGES FOR RENDER DEPLOYMENT

npm install cloudinary multer-storage-cloudinary

# OR add to your package.json dependencies:
{
  "cloudinary": "^1.41.0",
  "multer-storage-cloudinary": "^4.0.0"
}

# DEPLOYMENT STEPS FOR RENDER:

1. SETUP CLOUDINARY:
   - Sign up at https://cloudinary.com (free tier available)
   - Get your cloud_name, api_key, and api_secret from dashboard
   - Add these to Render environment variables

2. UPDATE YOUR CODE:
   - Replace uploadController.js with uploadController-RENDER-READY.js
   - Install cloudinary packages
   - Update upload route if needed

3. RENDER ENVIRONMENT VARIABLES:
   Set these in your Render dashboard:
   - NODE_ENV=production
   - CLOUDINARY_CLOUD_NAME=your_cloud_name
   - CLOUDINARY_API_KEY=your_api_key  
   - CLOUDINARY_API_SECRET=your_api_secret
   - All your existing MongoDB, JWT, etc. variables

4. BENEFITS OF CLOUDINARY:
   ✅ Persistent file storage
   ✅ CDN for fast file delivery
   ✅ Automatic image optimization
   ✅ Works with Render, Vercel, Heroku, etc.
   ✅ Free tier: 25GB storage, 25GB bandwidth/month
